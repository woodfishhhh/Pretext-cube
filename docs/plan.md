# 超立方体单屏展陈页方案

## Summary

- 将当前空白的 Vue/Vite 首页改造成一个几乎纯单屏的沉浸式展陈页，不介绍 pretext 本身，只把它作为
  核心排版引擎大量使用。
- 页面主舞台为黑底科幻展厅：中心是可拖拽、带惯性缓动的实体玻璃感超立方体；周围是一篇关于超立方体
  的激进多语长文。
- 交互职责明确分离：拖拽/触摸横向操作负责旋转雕塑，滚轮/纵向手势负责推进文章；两者都会触发实时绕
  排。
- 视觉取向偏装置而非纯阅读：idle 状态下文字半可读、分层悬浮；交互时局部可读性显著提升。

## Implementation Changes

- 舞台与视觉
  - 做满视口的单屏舞台，移除传统导航、卡片区块和说明区，只保留标题、极短副标题、交互提示。
  - 中心对象使用 Three.js/WebGL 渲染“玻璃感超立方体”投影，包含缓慢自转、拖拽旋转、松手惯性、
    冷色边缘辉光和轻度折射/透视感。
  - 文字层围绕中心物体分布，采用“主阅读带 + 环境碎片层”结构：左右两侧承担主要可读段落，上下与
    斜向区域承担更弱、更碎的悬浮文本，保证“更偏装置”的效果。
- pretext 排版内核
  - 文章内容维护为一篇连续多语长文，但单屏只显示当前“阅读窗口”内的一段文本。
  - 文本或字体变化时只执行一次 prepareWithSegments(...)；交互高频阶段不重新 prepare。
  - 每次超立方体姿态变化后，先把当前 3D/4D 投影转成 2D 轮廓包络，再按扫描线生成每一行的可用宽
    度。
  - 使用 layoutNextLine(...) 逐行求解，而不是只用 layout(...) 算总高度；这样每一行都能根据当前
    轮廓产生不同宽度，真正实现“绕物体实时重排”。
  - 重排触发源固定为：拖拽旋转、惯性缓动、文章推进、窗口 resize、移动端方向变化。
- 交互与阅读机制
  - 拖拽中心物体只改变姿态，不推进文章；滚轮/触摸纵向手势只推进文章，不承担旋转。
  - 文章推进时，新的文本窗口进入舞台，旧窗口离开；同一时刻始终保持单屏展陈，不出现传统长页正
    文。
  - idle 状态下文字透明度、模糊度和亮度分层，形成“半可读悬浮态”；用户交互时，靠近当前阅读通道
    的文字变清晰，外围碎片保持装置感。
  - 移动端保留同等强度的核心体验：触摸拖拽旋转、滑动推进文章、实时绕排依然存在；仅按视口尺寸减
    少同时显示的行数和碎片密度，不降级成交互静态页。

## Interfaces / Types

- 新增场景状态模型：
  - HypercubePose: 超立方体当前姿态、角速度、惯性状态
  - ProjectionEnvelope: 当前投影在各扫描线上的占位/留白信息
  - ArticleWindow: 连续长文的当前可见区间、推进偏移、方向
  - FlowLine: 单行文本的内容、宽度、位置、透明度层级
- 新增核心模块边界：
  - HypercubeStage: 负责 Three.js 场景、材质、姿态和投影数据输出
  - usePretextFlow: 负责 prepared 文本缓存、逐行宽度求解、可见窗口排版
  - OrbitTextLayer: 负责把排版结果渲染成主阅读带与外围碎片层
- 外部依赖变更：
  - 增加 three 作为 3D 渲染依赖；@chenglou/pretext 保持为主排版依赖，不引入替代型文本布局库。

## Test Plan

- 交互正确性
  - 桌面端拖拽旋转时，超立方体连续更新，周围文本无重叠、无明显跳行撕裂。
  - 滚轮推进文章时，文本窗口连续前进，阅读顺序不乱，旋转状态保持。
  - 移动端触摸旋转与纵向推进互不抢手势，页面不会误触整页滚动。
- 排版正确性
  - 中文、英文、数学术语、符号混排时，断行保持 grapheme 安全，不出现截断异常。
  - 超立方体姿态快速变化时，主阅读带始终避开中心投影，不穿模、不压字。
  - 视口 resize 和横竖屏切换后，文本重新布局稳定，无缓存错位。
- 性能与体验
  - 高频交互阶段保持平滑，至少达到“视觉连续、无明显卡顿”的体验目标；若帧率下降，应优先降低外围
    碎片层复杂度，而非关闭主绕排。
  - idle 态和交互态的可读性切换明显但不刺眼，避免过强闪烁。
  - 提供 prefers-reduced-motion 分支：保留单屏构图和绕排，但关闭强惯性与过度自动漂移。

## Assumptions

- 文章内容为一篇策展式“超立方体介绍长文”，由项目内静态文本提供，不接后端、不做 CMS。
- 页面目标是作品级 demo，不承担官网导航、产品转化、文档目录或 pretext 教学任务。
- 默认使用深色科幻基调、冷色高光、极少 UI chrome，不加入音频、表单、登录或额外功能面板。
- 当前仓库是轻量 Vue 3 + Vite 骨架，允许直接替换默认首页结构，并新增少量组件与渲染模块。
